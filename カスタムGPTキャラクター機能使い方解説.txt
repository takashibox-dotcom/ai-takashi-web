================================================================================
                            1. 基本的な使い方
================================================================================

【キャラクター作成の流れ】
┌─────────────────────────────────────┐
│ 1. [新規作成]ボタンをクリック                    │
│ 2. キャラクター名を入力（必須）                  │
│ 3. 各設定項目を自由に入力                        │
│ 4. [プレビュー]でシステムプロンプトを確認        │
│ 5. [キャラクター作成]で完了                      │
└─────────────────────────────────────┘

【キャラクター切り替え】
• [切替]ボタン → 使用したいキャラクターを選択
• 切り替え後は、そのキャラクターで会話が開始されます
• 各キャラクターの会話履歴は個別に保存されます

【キャラクター管理】
• [管理]ボタン → 編集・複製・削除が可能
• 検索機能で目的のキャラクターを素早く発見
• 使用回数・作成日などの統計情報も確認可能

================================================================================
                            2. キャラクター作成の詳細
================================================================================

【重要なポイント】
✓ キャラクター名以外はすべて任意入力
✓ どんな内容でも自由に設定可能
✓ 設定が多いほど、よりリアルなキャラクターに
✓ 空欄でも問題なく動作します

【作成のコツ】
1. まずは大まかな設定から始める
2. 具体的なエピソードや例を含める
3. プレビュー機能で動作確認する
4. 実際に使いながら調整していく

【システムプロンプト自動生成】
入力された設定から、AIが理解しやすい形式の
システムプロンプトが自動生成されます。
ユーザーは難しい技術的知識は不要です。

                            3. 各設定項目の解説
================================================================================

【🎭 性格・キャラクター設定】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
基本的な性格や特徴を自由に記述してください。

例：
• 優しくて知識豊富な先生。厳格だけど面倒見が良い
• フレンドリーで何でも相談できる相手
• ちょっと毒舌だけど的確なアドバイスをくれる
• 明るくて元気。いつも前向きで人を励ますのが上手
• 落ち着いていて冷静。論理的な思考を大切にする

【💬 話し方・口調】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
どのような話し方をするかを設定してください。

例：
• 関西弁で親しみやすく話す（「〜やで」「〜やんか」）
• 丁寧語で上品に話す（「〜ですね」「〜ましょう」）
• ため口でフランクに話す
• 敬語だけど親しみやすい口調
• 少し古風な話し方（「〜である」「〜なのじゃ」）

【🧠 専門分野・得意なこと】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
得意分野や専門知識を設定してください。

例：
• Python、JavaScript、データベース設計
• ビジネス戦略、財務分析、プレゼンテーション
• 小説執筆、創作技法、ストーリー構成
• 料理、生活の知恵、家事のコツ
• 心理学、カウンセリング、人間関係

【📝 応答スタイル】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
どのような形式で回答するかを設定してください。

例：
• 必ずサンプルコードを含める
• 段階的に分かりやすく説明する
• 簡潔で要点を整理した回答
• 具体例をたくさん使って説明
• 質問形式で相手の理解を確認しながら進める

【⚙️ その他の設定（任意）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• 背景・設定：キャラクターの背景ストーリー
• 決まり文句・口癖：特徴的な言い回し
• 挨拶の仕方：最初の挨拶スタイル
• カスタム指示：その他の特別な設定

================================================================================
                            6. よくある質問
================================================================================

【Q1】キャラクターはいくつまで作れますか？
【A1】最大100個まで作成可能です。十分な数のキャラクターを管理できます。

【Q2】設定項目は全て入力する必要がありますか？
【A2】キャラクター名以外はすべて任意です。空欄でも問題なく動作します。

【Q3】作成したキャラクターの設定を後から変更できますか？
【A3】はい。[管理]→[編集]から何度でも変更可能です。

【Q4】キャラクターを削除すると会話履歴も消えますか？
【A4】はい。キャラクターと関連する会話履歴も一緒に削除されます。

【Q5】AI_takashiキャラクターを削除できないのはなぜですか？
【A5】デフォルトキャラクターとして保護されているためです。これにより、
      システムの安定性が保たれます。

【Q6】どのような内容でも設定に書いて大丈夫ですか？
【A6】はい。システムは非常に柔軟で、どのような設定でも受け入れます。
      ただし、適切な内容の設定を推奨します。

【Q7】キャラクターの会話履歴は混在しませんか？
【A7】各キャラクターの会話履歴は完全に分離されているため、混在しません。

【Q8】システムプロンプトとは何ですか？
【A8】AIに「どのように振る舞うか」を指示する内部的な指令です。
      ユーザーの設定から自動生成されるため、技術的知識は不要です。

================================================================================
                          7. トラブルシューティング
================================================================================

【問題】キャラクターが作成できない
【解決策】
• キャラクター名が入力されているか確認
• 同じ名前のキャラクターが既に存在しないか確認
• 最大数（100個）に達していないか確認

【問題】思ったような応答が得られない
【解決策】
• 設定をより具体的に記述する
• プレビュー機能でシステムプロンプトを確認
• 何度か会話して学習させる
• 必要に応じて設定を調整する

【問題】キャラクターが表示されない
【解決策】
• アプリケーションを再起動する
• custom_characters.jsonファイルが正常か確認
• ログファイル（app.log）でエラーを確認

【問題】切り替えが反映されない
【解決策】
• [切替]ダイアログで正しく選択されているか確認
• 画面表示の更新を待つ
• 必要に応じてアプリケーションを再起動

◆ キャラクターデータファイル

形式：JSON形式
エンコーディング：UTF-8

◆ ログファイル
保存場所：\\ts-wxlea1\share\Automate_with_Python\AI\app.log
内容：システムの動作履歴、エラー情報、キャラクター操作ログ

【キャラクターデータの構造】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

custom_characters.jsonファイルには以下の情報が保存されています：

◆ 基本情報
• name：キャラクター名
• character_id：一意のキャラクターID（8桁の英数字）
• created_at：作成日時（ISO 8601形式）
• updated_at：最終更新日時（ISO 8601形式）
• usage_count：使用回数（会話した回数）
• is_default：デフォルトキャラクターフラグ（AI_takashiのみtrue）

◆ キャラクター設定項目
• personality：性格・キャラクター設定
• speaking_style：話し方・口調
• specialization：専門分野・得意なこと
• response_style：応答スタイル
• background：背景・設定
• constraints：制約事項
• catchphrase：決まり文句・口癖
• greeting：挨拶の仕方
• custom_instructions：その他のカスタム指示

◆ 会話履歴
• conversation_history：キャラクター別の会話ログ配列
  - timestamp：会話日時
  - user：ユーザーの発言
  - ai：AIの応答

【データファイルの例】
```json
[
  {
    "name": "AI_takashi",
    "character_id": "29d15281",
    "personality": "親しみやすく、気さくで話しかけやすい性格。",
    "speaking_style": "フレンドリーで親しみやすい口調。",
    "specialization": "一般的な質問への回答、様々な分野の基本的な知識。",
    "response_style": "分かりやすく丁寧な説明。",
    "background": "",
    "constraints": "",
    "catchphrase": "語尾は質問の場合は\"かね\" それ以外は\"なのだよ\"",
    "greeting": "質問あるかね?",
    "custom_instructions": "",
    "created_at": "2025-01-10T16:50:39.616733",
    "updated_at": "2025-01-10T17:07:18.090541",
    "usage_count": 1,
    "is_default": true,
    "conversation_history": [
      {
        "timestamp": "2025-01-10T17:05:09.831233",
        "user": "今日もお願いします",
        "ai": "もちろんですかね！今日もよろしくお願いするかね！"
      }
    ]
  }
]
```

【データ容量・制限】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• 最大キャラクター数：100個
• ファイルサイズ制限：なし（ただし大容量時は動作が重くなる可能性）
• 会話履歴：キャラクター別に無制限で保存
• 文字数制限：各設定項目に特別な制限なし

【バックアップ・復元方法】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

◆ 手動バックアップ
1. custom_characters.jsonファイルを別の場所にコピー
2. 定期的にファイルをUSBメモリやクラウドにバックアップ

◆ 自動バックアップ機能
• AI_takashiの[バックアップ管理]機能を使用
• 設定した間隔で自動的にバックアップを作成
• 復元も可能

◆ 復元方法
1. バックアップしたcustom_characters.jsonファイルを元の場所に上書き
2. AI_takashiを再起動
3. キャラクターが復元されていることを確認

【データ保護に関する注意事項】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• 重要なキャラクター設定は必ずバックアップを取る
• ファイルの直接編集は推奨しません（データ破損の可能性）
• システム更新前にはバックアップを作成
• 複数人で同じファイルを使用する場合は競合に注意

【トラブル時のデータ復旧】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. custom_characters.jsonが破損した場合
   → バックアップファイルから復元
   → ファイルが無い場合はAI_takashiが自動的に新規作成

2. キャラクターが消えた場合
   → app.logファイルでエラーログを確認
   → バックアップから復元
   → 必要に応じて手動で再作成

3. 会話履歴が消えた場合
   → バックアップファイルから復元
   → エクスポート機能で事前保存していれば復旧可能
